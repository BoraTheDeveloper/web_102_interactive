Step 2 of 6 · Theory

Events, handlers & state

Read the foundations. Take your time — examples come next.

Goal

Understand the JavaScript behind React state — events, event handlers, useState, and the immutable array updates (spread and .map()) React relies on.

An event is something the user does — clicking a button, typing in an input, or submitting a form.

In React, an event handler is a function that runs after an event happens.

function handleClick() {
  console.log("Button clicked");
}

State stores data that can change on the screen. Created with useState; when it changes, React updates the UI to match.

const [count, setCount] = useState(0);

For arrays, React expects a new array. Spread ...tasks copies the old values into a new one:

const tasks = ["Study JS", "Practice React"];
const newTasks = [...tasks, "Build planner"];

State & User Interaction

Events, State, and Array Updates

Goal

Understand the JavaScript behind React state and user interaction — events, event handlers, useState, and the immutable array updates (spread and .map()) that React relies on.

01

Theory

An event is something the user does — for example clicking a button, typing in an input, or submitting a form.

In React, an event handler is a function that runs after an event happens.

function handleClick() {
  console.log("Button clicked");
}

State stores data that can change on the screen. In React we create it with useState.

const [count, setCount] = useState(0);

When state changes, React automatically updates the UI to match.

For arrays, React expects you to create a new array instead of changing the old one directly. The spread syntax ...tasks copies the old values into a new array.

const tasks = ["Study JS", "Practice React"];
const newTasks = [...tasks, "Build planner"];

.map() runs a function on every item and returns a new array with the results. In React, .map() is how you turn a list of data into a list of elements.
